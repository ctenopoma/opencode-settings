## 1. 背景
レガシーの `COMMON /grid/ nx, ny, dx, dy` は 12 個のサブルーチンから直接参照され、`BLOCK DATA` で初期化されている。移行先で状態の持たせ方を決める必要がある。

## 2. 何が決められないか
グローバルのまま移すと Rust/Python 境界でスレッド安全性とテスト容易性を損なう。一方で全サブルーチンの引数に渡すと署名が大きく変わり、レガシーとの差分追跡が難しくなる。挙動は不変が前提だが、構造選択は設計判断であり AI 単独では決め切れない。

## 3. 選択肢
- **A: グローバルシングルトンを維持** — 差分最小。だが並列化・テストが困難。
- **B: `Grid` 構造体に閉じ込め、明示的に引き回す** — テスト容易・並列安全。署名変更は大きいが機械的。
- **C: スレッドローカル状態** — 当面の差分は小さいが、将来の並列設計を縛る。

## 4. 推奨と根拠
**B**。本プロジェクトはリファクタ込みの移行であり、golden 比較で挙動不変は担保できる。長期保守性と並列化余地を取るべき。
