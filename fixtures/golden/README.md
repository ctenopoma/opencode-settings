# fixtures/golden — レガシー基準入出力

検証ゲートの基準。**採取方法は問わない**（A: 既存バイナリをブラックボックス実行 / B: 過去の回帰出力流用 / C: Intel ifx で採取）。
`tools/verify/compare.py` はここに置かれた基準と新実装出力を数値比較するだけなので、どの方法で採っても動く。

## 配置
```
fixtures/golden/<unit-id>/
├─ input/        # レガシーに与えた入力（再現用）
├─ expected/     # レガシーの基準出力（= 正解）
└─ actual/       # 新実装の出力（dev/qa が生成。git 管理外でよい）
```

## 採取のコツ
- 入力は代表 + 端（境界値・特異点・空・最大規模）を含める。
- 期待出力は**桁を落とさず**保存（丸めると許容誤差判定が無意味になる）。
- 単位・スケールを README にメモ。許容誤差(rtol/atol)は設計書側で定義する。
